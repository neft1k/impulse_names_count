# count-names

Консольная утилита: читает файл с именами (по одному на строку) и печатает частоту каждого имени.

## Запуск

Сборка:

```bash
make build
```

Из файла:

```bash
./count-names names.txt
```

Из stdin:

```bash
cat names.txt | ./count-names
```

Сортировка по алфавиту вместо частоты:

```bash
./count-names -alpha names.txt
```

Только топ-10 самых частых:

```bash
./count-names -top 10 names.txt
```

## Тесты и линтер

Юнит и интеграционные тесты:

```bash
make test
```

Бенчмарки:

```bash
make bench
```

Линтер:

```bash
make lint
```

## Ответы на вопросы из задания

1. Что делать, если файл очень большой?
Читать построчно через bufio.Scanner — в памяти держится только map[string]int, размер которой определяется числом уникальных имён, а не общим числом строк.

2. Какие тесты можно написать?
Юнит-тесты на Count (пустой ввод, дубли, пробелы, юникод, ошибки чтения) и Write (обе сортировки, top-N, тай-брейк), интеграционные тесты на CLI (stdin, файл, флаги, ошибки).

3. Можем ли выводить в порядке убывания частоты?
Да, это поведение по умолчанию. При равных частотах применяется вторичная сортировка по алфавиту — вывод детерминированный.
